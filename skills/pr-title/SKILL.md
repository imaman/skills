Comeup with a PR title for all the changes in the current branch.

## Workflow:

(1) Take a look at the most recent 40 PR titles. Do do that use git log and filter for commits which are PRs: essentially any commit title ends with (#<PR-NUMBER>). Use this list for getting a sense of how PR titles look like in this repo.
(2) Run `dcc diff` for getting these diffs.
(3) Based on (1) and (2) devise four possible PR title. Provide a 1-setnence ratiole for each and then picking the winning one.
